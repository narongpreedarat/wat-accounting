# คู่มือระบบสมุดบัญชีวัดออนไลน์

> สำหรับเจ้านายที่เพิ่งเริ่มต้น — ทำตามทีละขั้น ไม่ต้องมีความรู้เขียนโปรแกรมมาก่อน
> ทั้งหมดนี้ **ฟรี** ไม่มีค่าใช้จ่าย

ระบบนี้ให้หลายวัดเข้ามาใช้ร่วมกันได้ โดย**ข้อมูลแต่ละวัดแยกกันสนิท** วัดอื่นมองไม่เห็นข้อมูลของเรา เพราะใช้ระบบล็อกอินและกฎความปลอดภัยของ Supabase

---

## ภาพรวม 4 ขั้นตอน

1. สมัคร Supabase (ที่เก็บข้อมูลกลาง) — ฟรี
2. สร้างตารางข้อมูลด้วยสคริปต์ที่เตรียมไว้ให้ — คัดลอกวาง กดปุ่มเดียว
3. นำค่าเชื่อมต่อมาใส่ในไฟล์ config
4. อัปโหลดขึ้น GitHub แล้วเปิดให้เว็บทำงานออนไลน์ — ฟรี

เวลาที่ใช้ทั้งหมดประมาณ 30–45 นาที

---

## ขั้นที่ 1 — สมัคร Supabase (ที่เก็บข้อมูลกลาง)

Supabase คือบริการฐานข้อมูลออนไลน์ฟรี ที่มาพร้อมระบบล็อกอินและการรักษาความปลอดภัยในตัว เหมาะกับงานนี้มาก

1. เปิดเว็บ https://supabase.com แล้วกด **Start your project**
2. สมัครด้วยบัญชี GitHub ของเจ้านาย (กด Continue with GitHub) — สะดวกที่สุดเพราะมีบัญชี GitHub อยู่แล้ว
3. เมื่อเข้าได้แล้ว กด **New project**
   - **Name**: ตั้งชื่อเช่น `wat-accounting`
   - **Database Password**: ตั้งรหัสผ่านฐานข้อมูล (จดเก็บไว้ให้ดี แต่ไม่ต้องใช้บ่อย)
   - **Region**: เลือก `Southeast Asia (Singapore)` เพราะใกล้ไทยที่สุด เร็วกว่า
4. กด **Create new project** แล้วรอประมาณ 1–2 นาที ให้ระบบสร้างเสร็จ

> หมายเหตุ: บัญชีฟรีของ Supabase รองรับได้หลายวัดสบาย ๆ (พื้นที่ 500 MB, ผู้ใช้ได้ถึง 50,000 คน) เพียงพอสำหรับวัดเป็นร้อยวัด

---

## ขั้นที่ 2 — สร้างตารางข้อมูล (สำคัญที่สุด)

ขั้นนี้คือการสร้าง "ช่องเก็บข้อมูล" และ "กฎแยกข้อมูลแต่ละวัด" เราเตรียมสคริปต์ให้แล้ว เจ้านายแค่คัดลอกวาง

1. ในหน้า Supabase ของโปรเจกต์ มองเมนูด้านซ้าย หาไอคอน **SQL Editor** (รูปฐานข้อมูล)
2. กด **New query** (ปุ่มขวาบน)
3. เปิดไฟล์ `database_setup.sql` ที่อยู่ในโฟลเดอร์นี้ คัดลอก**ทั้งหมด**
4. วางลงในช่อง SQL Editor
5. กดปุ่ม **Run** (มุมขวาล่าง หรือกด Ctrl+Enter)
6. ถ้าขึ้นข้อความเขียว "Success" แปลว่าสำเร็จ — ตารางและกฎความปลอดภัยถูกสร้างเรียบร้อย

> สิ่งที่สคริปต์นี้ทำ: สร้างตาราง 4 ตาราง (ข้อมูลวัด, รายรับ-รายจ่าย, ทรัพย์สิน, เงินฝาก) และตั้งกฎ "Row Level Security" ที่บังคับว่าแต่ละวัดเห็นได้เฉพาะข้อมูลของตัวเองเท่านั้น — นี่คือหัวใจที่ทำให้ข้อมูลไม่ปนกัน

### ตั้งค่าระบบล็อกอิน (ทำครั้งเดียว)

เพื่อให้สมัครและล็อกอินได้ง่ายในช่วงเริ่มต้น แนะนำให้ปิดการยืนยันอีเมลก่อน:

1. เมนูซ้าย ไปที่ **Authentication** → **Sign In / Providers** (หรือ **Settings**)
2. หาหัวข้อ **Email** ตรวจว่าเปิดใช้งาน (Enabled)
3. หา **Confirm email** แล้ว**ปิด** (toggle ออก) — เพื่อให้สมัครแล้วล็อกอินได้ทันที ไม่ต้องรอเมลยืนยัน
4. กด Save

> ภายหลังถ้าต้องการความปลอดภัยสูงขึ้น ค่อยเปิด Confirm email กลับมาได้

---

## ขั้นที่ 3 — นำค่าเชื่อมต่อมาใส่ในไฟล์ config

ตอนนี้เราต้องบอกเว็บของเราว่า "ฐานข้อมูลอยู่ที่ไหน" ด้วยค่า 2 ตัว

1. ในหน้า Supabase เมนูซ้ายล่าง ไปที่ **Project Settings** (ไอคอนเฟือง) → **API Keys** (หรือ **Data API**)
2. เจ้านายจะเห็นค่า 2 อย่าง:
   - **Project URL** — หน้าตาเหมือน `https://abcdxyz.supabase.co`
   - **anon public** key — เป็นข้อความยาว ๆ ขึ้นต้นด้วย `eyJ...`
3. เปิดไฟล์ `js/config.js` ในโปรแกรม VS Code
4. แทนที่ค่า 2 บรรทัด:

```javascript
const SUPABASE_URL = 'https://abcdxyz.supabase.co';        // ← วาง Project URL
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1...';           // ← วาง anon public key
```

5. กด **บันทึก** (Ctrl+S)

> ค่า `anon public` key นี้เปิดเผยได้ ไม่เป็นความลับ เพราะกฎความปลอดภัยที่เราตั้งในขั้นที่ 2 จะคอยป้องกันข้อมูลอยู่แล้ว (ห้ามใช้ค่า `service_role` key เด็ดขาด — ตัวนั้นเป็นความลับ)

### ทดสอบในเครื่องก่อน (ไม่บังคับ แต่แนะนำ)

เปิดไฟล์ `index.html` ด้วยเบราว์เซอร์ (ดับเบิลคลิกที่ไฟล์ หรือลากไปวางในหน้าต่าง Chrome)

- ถ้าเห็นหน้าล็อกอิน → ตั้งค่าถูกต้องแล้ว
- ลองกด "สมัครวัดใหม่" กรอกชื่อวัด อีเมล รหัสผ่าน แล้วลองเข้าใช้งาน

> เกี่ยวกับ error ที่เจ้านายเคยเจอ ("running scripts is disabled"): ระบบนี้**ไม่ต้องใช้ npm เลย** จึงไม่เจอปัญหานั้นอีก เปิดไฟล์ HTML ได้ตรง ๆ

---

## ขั้นที่ 4 — อัปโหลดขึ้น GitHub และเปิดเว็บออนไลน์

ตอนนี้เว็บอยู่ในเครื่องเรา เราจะเอาขึ้น GitHub เพื่อให้ทุกวัดเข้าถึงผ่านอินเทอร์เน็ตได้ (ใช้บริการ GitHub Pages ฟรี)

### วิธีที่ง่ายที่สุด (ผ่านหน้าเว็บ ไม่ต้องใช้คำสั่ง)

1. เข้า https://github.com แล้วล็อกอิน
2. กดปุ่ม **+** มุมขวาบน → **New repository**
   - **Repository name**: เช่น `wat-accounting`
   - เลือก **Public** (จำเป็นสำหรับ GitHub Pages ฟรี)
   - กด **Create repository**
3. ในหน้า repo ใหม่ กดลิงก์ **uploading an existing file**
4. ลากไฟล์**ทั้งหมด**ในโฟลเดอร์นี้ (index.html, โฟลเดอร์ css, โฟลเดอร์ js, database_setup.sql) เข้าไปวาง
   - หมายเหตุ: ลากทั้งโฟลเดอร์ css และ js เข้าไปด้วย เพื่อรักษาโครงสร้าง
5. กด **Commit changes**

### เปิด GitHub Pages

1. ในหน้า repo ไปที่ **Settings** (เมนูบนของ repo)
2. เมนูซ้าย เลือก **Pages**
3. หัวข้อ **Branch** เลือก `main` แล้วกด **Save**
4. รอประมาณ 1–2 นาที จากนั้นรีเฟรชหน้า จะเห็นลิงก์เว็บของเจ้านาย หน้าตาเช่น:
   `https://ชื่อผู้ใช้.github.io/wat-accounting/`
5. เปิดลิงก์นั้น — นี่คือเว็บออนไลน์ที่ทุกวัดเข้าใช้ได้แล้ว!

> ส่งลิงก์นี้ให้วัดต่าง ๆ แต่ละวัดสมัครบัญชีของตัวเอง ข้อมูลจะแยกกันอัตโนมัติ

---

## การใช้งานประจำวัน

- **แต่ละวัดสมัครบัญชีของตัวเอง** (1 อีเมล = 1 วัด) ด้วยปุ่ม "สมัครวัดใหม่"
- เข้าเมนู **ข้อมูลวัด** กรอกชื่อวัด ที่อยู่ เจ้าอาวาส ไวยาวัจกร
- บันทึกเงินเข้า-ออกที่เมนู **รายรับ-รายจ่ายรายวัน**
- ดูสรุปแยกหมวดที่เมนู **บัญชีแยกรายรับ / รายจ่าย**
- บันทึกทรัพย์สินที่ **ทะเบียนทรัพย์สิน** และเงินฝากที่ **บัญชีเงินฝาก**
- สิ้นปีไปที่ **รายงานงบประจำปี** กด **พิมพ์รายงาน** เพื่อส่ง พศ.

## คำถามที่พบบ่อย

**ถาม: ปลอดภัยจริงหรือ วัดอื่นเห็นข้อมูลเราไหม?**
ตอบ: ไม่เห็นครับ กฎ Row Level Security ที่ตั้งในขั้นที่ 2 บังคับให้ฐานข้อมูลส่งคืนเฉพาะแถวที่เป็นของผู้ล็อกอินอยู่เท่านั้น แม้จะใช้ฐานข้อมูลและเว็บเดียวกัน

**ถาม: ฟรีจริงไหม มีวันหมดอายุไหม?**
ตอบ: ทั้ง Supabase และ GitHub Pages มีแพ็กเกจฟรีถาวร เพียงพอสำหรับการใช้งานจริงของวัดหลายแห่ง หากในอนาคตข้อมูลเยอะมากค่อยพิจารณาอัปเกรด

**ถาม: ถ้าลืมรหัสผ่านวัด?**
ตอบ: ใช้ฟังก์ชันรีเซ็ตรหัสผ่านของ Supabase หรือผู้ดูแล (เจ้านาย) จัดการได้ในเมนู Authentication ของ Supabase

**ถาม: อยากแก้ไขหน้าตาเว็บ?**
ตอบ: แก้ไฟล์ `css/style.css` (สี ขนาด) หรือ `js/app.js` (เนื้อหา) แล้วอัปโหลดทับขึ้น GitHub ใหม่

**ถาม: error "running scripts is disabled" ที่เคยเจอใน VS Code?**
ตอบ: นั่นเกิดตอนพยายามรัน npm ระบบนี้ไม่ใช้ npm จึงไม่เจออีก แต่ถ้าอยากแก้ที่ต้นเหตุไว้ใช้งานอื่น เปิด PowerShell แบบ Run as Administrator แล้วพิมพ์: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` แล้วกด Y

## โครงสร้างไฟล์

```
wat-accounting/
├── index.html            ← หน้าเว็บหลัก (เปิดไฟล์นี้)
├── css/
│   └── style.css         ← รูปแบบหน้าตา สี ฟอนต์
├── js/
│   ├── config.js         ← ⭐ ไฟล์เดียวที่เจ้านายต้องแก้ (ใส่ค่า Supabase)
│   └── app.js            ← โค้ดหลัก (ไม่ต้องแก้)
├── database_setup.sql    ← สคริปต์สร้างฐานข้อมูล (ใช้ในขั้นที่ 2)
└── README.md             ← คู่มือนี้
```

หากติดขัดขั้นตอนใด แจ้งมาได้เลย ผมช่วยแก้ทีละจุดครับ
